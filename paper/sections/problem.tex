\section{问题与动机}

Fisher、Lynch 和 Paterson 的论文研究的是异步消息传递系统中的容错共识问题。“共识”是指一组相互通信的进程在各自持有初始输入的情况下，最终对同一个输出值作出决定。在该论文的基本设定中，每个进程的输入值为 $0$ 或 $1$，协议的目标是在存在有限故障的情况下，使所有作出决定的非故障进程决定同一个二进制值。

这一问题的重要之处在于，共识是许多分布式系统维持一致状态的基础机制。论文在引言中以分布式数据库的事务提交问题为例：一笔交易可能由多个数据管理进程共同参与，系统最终必须一致地决定提交交易结果还是放弃交易结果。如果一部分进程提交而另一部分进程放弃，数据库副本就会进入互相矛盾的状态。因此，共识并不是一个纯粹形式化的理论游戏，而是分布式数据处理、分布式文件管理和容错应用中反复出现的核心需求。

在该论文的语境中，“协议”指每个进程共同遵守的一组确定性本地规则：进程如何根据初始输入设置状态，收到消息后如何更新本地状态，接下来向哪些进程发送什么消息，以及在什么条件下进入决定状态。论文将进程形式化为确定性自动机；因此，一旦给定进程当前状态和收到的消息，该进程的下一步行为就是确定的。不过，协议本身是确定性的，并不意味着系统只有唯一执行轨迹。由于系统是异步的，消息送达顺序和进程调度顺序仍然可以不同，同一个协议可能对应许多不同的合法执行。

一个共识协议通常需要满足以下三个要求。

\begin{itemize}
  \item \textbf{一致性}：不能有两个非故障进程决定不同的值。也就是只要进程已经作出决定，这些决定之间就不能互相矛盾。
  \item \textbf{非平凡性}：协议不能通过固定输出某个常数来规避问题。例如，无论输入如何都决定 $0$ 的协议虽然能够保持一致并终止，但它完全忽略了输入。该论文使用的非平凡性要求较弱：$0$ 和 $1$ 都必须在某些可达构型中可能成为决定值。
  \item \textbf{终止性}：在满足故障假设的合法执行中，协议最终必须产生决定。实际系统通常希望所有非故障进程最终都决定；该论文为了得到更强的不可能性结论，只需要考虑一个更弱的终止要求，即至少有某个进程最终进入决定状态。
\end{itemize}

该论文关注的是完全异步系统。在这种模型中，系统没有全局时钟，没有消息延迟上界，没有进程运行速度上界或下界，也没有可靠的故障检测器。消息系统本身是可靠的：消息不会丢失、损坏或重复，并且只要目标进程持续尝试接收，发送给它的消息最终会被投递。然而，消息可以被任意长时间地延迟，也可以乱序到达。论文考虑的故障也不是拜占庭故障：故障进程不会撒谎、伪造消息或恶意发送矛盾信息，而只是发生崩溃故障，即静默停止运行。论文进一步将故障数量限制为最多一个进程发生崩溃，这使得它研究的是一个非常基础但又具有代表性的容错共识设定。

上述模型中的核心困难在于：在完全异步系统里，进程无法区分另一个进程是已经崩溃，还是只是运行很慢，或者它发出的消息尚未被调度投递。举一个简单的例子。设三个数据管理进程 $P_1,P_2,P_3$ 共同处理一笔交易，每个进程先广播自己的初始意见，然后等待收到所有进程的意见，最后根据固定规则决定提交或放弃。如果没有进程崩溃，该协议可以成功，因为所有消息最终都会到达。但如果 $P_3$ 在发送消息之前崩溃，$P_1$ 和 $P_2$ 就会一直等待 $P_3$ 的消息，协议无法终止。反过来，如果协议允许进程过早忽略未响应的进程，那么在另一个合法执行中，被忽略的进程可能并未崩溃，只是运行较慢；此时过早决定又可能威胁一致性或非平凡性。

综上所述，该论文回答的问题是：

\begin{quote}
在一个完全异步、消息可靠、最多只有一个进程可能发生崩溃故障的分布式系统中，是否存在一个确定性共识协议，能够同时满足一致性、非平凡性和终止性？
\end{quote}
